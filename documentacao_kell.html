Visão Geral da Arquitetura

Solução no padrão cliente-servidor. O Frontend Angular 20 se comunica com o Backend Node.js via HTTP REST, protegido por autenticação JWT. O banco de dados PostgreSQL é acessado exclusivamente pelo servidor através do Sequelize ORM.

🖥️ Backend — BackEndKell

  • Node.js + Express 5
  • Sequelize 6 + PostgreSQL
  • JWT + bcryptjs
  • CORS + dotenv
  • Porta padrão: :3001

🌐 Frontend — FrontEndKell

  • Angular 20 + TypeScript
  • RxJS (programação reativa)
  • jsPDF + html2canvas (PDF)
  • jwt-decode
  • Route Guards + Interceptors

Diagrama — Fluxo de Autenticação (Login)

Sequência completa desde o preenchimento do formulário até o redirecionamento pós-login.

sequenceDiagram
    actor Usuario as Usuário
    participant Login as Angular /login
    participant API as POST /auth/login
    participant DB as PostgreSQL

    Usuario->>Login: Preenche e-mail e senha
    Login->>API: POST /auth/login { email, password }
    API->>DB: SELECT * FROM users WHERE email = ?
    DB-->>API: { id, nome, passwordHash, roleId }
    API->>API: bcrypt.compare(password, hash)

    alt Credenciais inválidas
        API-->>Login: 401 Unauthorized
        Login-->>Usuario: Mensagem de erro
    else Credenciais corretas
        API->>API: jwt.sign({ id, roleId })
        API-->>Login: 200 OK { token, user }
        Login->>Login: Salva token no LocalStorage
        Login-->>Usuario: Redireciona ao Dashboard
    end

Diagrama — Fluxo de Criação de OS

Sequência crítica de criação de uma Ordem de Serviço, do frontend ao banco de dados.

sequenceDiagram
    actor Op as Operador
    participant UI as Angular /criacao-os
    participant Int as auth.interceptor
    participant MW as Middleware JWT
    participant Ctrl as ordemServico Controller
    participant DB as PostgreSQL

    Op->>UI: Preenche OS (confeccao, produtos, qtd)
    UI->>Int: POST /ordens
    Int->>Int: Injeta Bearer token no header
    Int->>MW: Requisicao autenticada
    MW->>MW: jwt.verify(token)

    alt Token invalido ou expirado
        MW-->>UI: 403 Forbidden
        UI-->>Op: Redireciona para login
    else Token valido
        MW->>Ctrl: Processa requisicao
        Ctrl->>DB: INSERT INTO ordem_servicos
        DB-->>Ctrl: Retorna ordemId
        loop Para cada item
            Ctrl->>DB: INSERT INTO ordem_items
        end
        Ctrl-->>UI: 201 Created { ordemId }
        UI-->>Op: OS criada com sucesso
    end

Fluxo de Segurança (Resumo)

  1. Usuário acessa o sistema e faz login na tela /login.
  2. Angular envia credenciais para POST /auth/login.
  3. Backend valida e-mail + senha (bcrypt) e retorna um token JWT assinado.
  4. Token é salvo no LocalStorage pelo Angular.
  5. O auth.guard bloqueia rotas protegidas para usuários sem token.
  6. O auth.interceptor injeta o token em toda requisição HTTP subsequente.
  7. O Middleware do Backend valida o token em cada endpoint protegido.
  8. Em caso de token inválido ou expirado, o servidor retorna 403 e o Angular realiza logout.
